# 싱글톤 패턴 (Singleton Pattern)
: 단 하나의 유일한 인스턴스만 존재하도록 보장하는 패턴


### 기본 구조
- 생성자를 외부에서 호출하지 못하도록 함 (접근 제어 설정)
- 내부에서 단 하나의 인스턴스 생성
- 정적(Static)으로 인스턴스 제공

```swift
final class Singleton {
	static let shared = Singleton()
	
	private init() { }
}
```


### 사용하는 곳
- 앱 전역에서 인스턴스가 하나만 존재해야하는 상태
- 공유 자원을 관리할 때
- 시스템 성격의 서비스 (Analytics)


## 싱글톤이 안티패턴인 이유
- OCP 원칙 위반: 항상 자체 인스턴스를 반환하여 확장에 열려있는 경우가 없음.
- SRP 원칙 위반: 인스턴스 생성, 실제 작업 수행 2가지 책임을 가지고 있어서 위반
- 테스트하기 어려움: 전역 상태라서 의존성을 숨겨 테스트하기 어려움
