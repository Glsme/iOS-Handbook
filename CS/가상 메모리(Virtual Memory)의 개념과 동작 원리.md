# 가상 메모리
실제 물리적 RAM 용량보다 더 큰 메모리 공간을 사용하는 것처럼 보이게 하는 운영체제의 메모리 관리 기법

## 가상 메모리 사용 이유
1. RAM보다 큰 프로그램 실행 가능
2. 프로세스 메모리 보호 
   (각 프로세스는 자기만의 가상 주소 공간을 갖기 때문에 다른 프로그램의 메모리를 함부로 읽거나 쓰기 어려움)
3. 메모리 관리 용이

## MMU(Memory Management Unit)
CPU 안에 있는 하드웨어로 물리 주소와 가상 주소를 연결.
- 가상 주소를 물리 주소로 변환
- RAM을 여러 **페이지(page)** 로 나누어 각 페이지를 하나의 독립된 항목으로 처리
  \*페이지(page): 가상 메모리를 일정 크기로 나눈 단위

## 페이징 (Paging)
운영체제에서 메모리를 효율적으로 관리하기 위해 프로그램을 **페이지(Page)** 로 나누고, 물리 메모리를 **프레임(Frame)** 로 나누어 비연속적으로 적재하여 실행하는 기법
\*프레임(frame): 물리 메모리를 일정 크기로 나눈 단위

### 페이지 테이블(Page Table)
OS가 관리하는 어떤 페이지가 어떤 프레임에 위치해 있는지 기록한 매핑표
- 가상 페이지 번호 (VPN) → 물리 페이지 번호 (PFN) 을 기록

<주소 변환 예시>
`[페이지 번호 | 페이지 내부 오프셋]` 일 때

1. 페이지 번호로 페이지 테이블 검색
2. 해당 페이지가 RAM에 있으면 → 물리 프레임 번호 획득
3. 물리 프레임 번호 + 오프셋 → 물리 주소 완성
4. 페이지가 없다면? → 페이지 폴트 (page faults)

### 페이지 폴트(page faults)
프로그램에 접근하려는 페이지가 RAM에 없을 때 발생하는 이벤트

페이지 폴트 발생 시 흐름
1. CPU가 메모리 접근 시도
2. MMU가 페이지 테이블 확인
3. RAM에 없다면 페이지 폴트(interrupt) 발생
4. OS가 디스크(스왑 영역)에서 해당 페이지를 RAM에 로드
5. 페이지 테이블 업데이트
6. 원래 명령 재실행

### TLB (Translation Lookaside Buffer)
CPU가 가상 주소를 물리 주소로 변환하는 속도를 높이기 위한 **주소 변환 캐시**


페이지 폴트가 자주 일어날수록 운영 체제의 성능이 많이 저하되기 때문에 페이지 폴트가 일어나지 않도록 하는 것이 중요 → 페이지 교체 기법 사용

### 페이지 교체 기법
메모리를 관리하는 운영체제에서 필요한 페이지가 주기억장치에 적재되지 않았을 시(페이지 부재) 어떤 페이지 프레임을 선택하여 교체할 것인지 결정하는 방법

1. **OPT** - Optimal : 앞으로 가장 오랫동안 사용되지 않을 페이지 교체
2. **FIFO** - First In First Out
3. **LRU** - Least **_Recently_** Used : 가장 오랫동안 사용되지 않은 페이지 교체
4. **LFU** - Least **_Frequently_** Used : 참조 횟수가 가장 작은 페이지 교체
5. **MFU** - Most Frequently used : 참조 횟수가 가장 많은 페이지 교체
6. **NUR** - Not Used Recently : 최근에 사용하지 않은 페이지 교체

## 가상 메모리 단점
- 쓰레싱 (Thrashing): 실행하려는 프로그램이 너무 많아서 CPU가 실제 연산보다 페이지 교체에 더 많은 시간을 사용하는 상태
