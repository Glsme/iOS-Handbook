# 자료구조 종류
### 배열 (Array)
- 동일한 타입의 데이터들을 연속된 메모리 공간에 저장하는 자료 구조
- 배열을 구성하는 각각의 값을 요소(Element) 
- 인덱스(index)로 배열 내에서 요소의 위치를 나타냄
 - 장점
	 - 인덱스를 통해 배열의 요소에 빠르게 접근 가능
	 - 동일한 데이터 타입의 요소가 연속적으로 저장되어 메모리를 효율적으로 사용 가능
	 - 단일 변수를 통해 대량의 데이터를 관리해 데이터 처리 과정을 간결하게 처리 가능
 - 단점
	 - 생성 시점에 크기가 정해지며, 이후 크기를 동적으로 변경하기 어려움
	 - 배열에 데이터를 삽입, 삭제 시 시간이 많이 소요됨
	 - 배열의 크기를 데이터의 최대 예상 개수에 맞추어 할당하면, 실제 사용되지 않는 공간이 발생할 수 있어 메모리가 낭비될 수 있음

### 연결 리스트 (Linked List)
- 데이터와 다음 데이터의 주소(포인터, Pointer)를 함께 가진 노드(Node)들이 연결된 자료 구조
- 노드는 다음 노드의 위치를 가리킴
- 메모리에 연속적으로 저장될 필요가 없음
- 장점
	- 필요에 따라 메모리를 할당하여 크기를 동적으로 조절 가능
	- 삽입 / 삭제가 빠름 (O(1), 위치만 알면 포인터만 변경)
	- 메모리 낭비 감소 (배열처럼 남는 공간이 발생하지 않음)
- 단점
	- 인덱스 접근이 불가능하여 탐색 속도가 느림 (O(n))
	- 데이터와 포인터를 저장하여 추가적인 메모리 공간이 필요
	- 메모리 위치가 흩어져있어 캐시 효율이 낮음 (CPU 캐시 활용이 어려움)
- 종류
	- 단일 연결 리스트 (Singly Linked List): 다음 노드만 참조
	- 이중 연결 리스트 (Doubly Linked List): 이전 + 다음 노드 참조
	- 원형 연결 리스트 (Circular Linked List): 마지막 노드가 처음을 가리킴


### 스택 (Stack)
- 후입선출(LIFO) 형태의 자료 구조
- 장점
	- 구조가 단순하고 구현이 쉬움
	- 스택의 상단에 요소를 추가하거나 제거하기 때문에 속도가 빠름
- 단점
	- Top 이외엔 접근 불가능
	- 스택의 크기가 고정되면 그 크기를 초과하는 데이터 저장 불가능

### 큐(Queue)
- 선입선출(FIFO) 형태의 자료 구조
- 장점
	- 자료의 순서를 유지하면서 요소를 관리하여 시간에 따른 데이터 처리에 적합
	- 양 끝에서 접근이 가능하여 삽입 / 삭제 개념이 명확
- 단점
	- 중간 요소 접근이 불가능하여 탐색 / 검색에 부적합
	- 배열 기반의 큐일 경우 dequeue 시 시간 복잡도 증가 -> 원형 큐 or 덱으로 해결 가능
- 종류
	- 일반 큐: FIFO 구조
	- 원형 큐 (Circular Queue): 앞 공간을 재사용하며, 배열 기반 큐의 단점을 보완
	- 우선순위 큐 (Priority Queue): 우선 순위가 높은 데이터 먼저 처리(Not FIFO). 보통 힙(Heap)으로 구현
	- 덱 (Deque): 앞 / 뒤 로 삽입 또는 삭제가 가능하며 큐와 스택의 성격을 동시에 가지고 있음.

### 해시 테이블 (Hash Table)
- 키(Key)를 해시 함수로 변환하여, 그 결과를 인덱스로 사용하여 값을 저장 / 조회하는 자료 구조
	- 해시 함수: 데이터를 효율적으로 관리하기 위해 데이터를 수학적 연산을 통해 고정된 길이의 데이터로 매핑하는 함수
- 해시 테이블 동작 원리
	- 해시 함수로 Key를 인덱스로 변환 (항상 같은 키는 같은 hash값을 가짐)
	- hash 값을 배열 인덱스로 사용
	- 충돌: 서로 다른 key가 같은 index를 가리킬 경우
		- 체이닝(Chaining): 같은 인덱스에 연결 리스트 저장
		- 개방 주소법(Open Addressing): 빈 공간을 찾아 이동 및 저장
- 장점
	- 매우 빠른 탐색 속도
	- 필요한 만큼의 메모리만을 사용하여 데이터를 저장하기 때문에 공간 효율성이 높음
- 단점
	- 충돌 발생 가능 -> 성능 저하
	- 순서 보장 불가능


### 트리 (Tree)
- 계층적(부모 - 자식)관계를 표현하는 비선형 자료 구조.
- 장점
	- 계층 구조 표현에 최적
	- 탐색에 효율적 (O(log n))
	- 데이터 삽입 / 삭제에 유연
- 단점
	- 구현이 복잡
	- 자식 노드에 대한 참조를 저장해야 하여 메모리 사용량이 증가

### 그래프 (Graph)
- 정점(Vertex)과 간선(Edge)으로 이루어진 자료 구조
- 데이터 사이의 관계를 표현하는데 특화
- 장점
	- 복잡한 관계를 표현하는데 유용
	- 새로운 요소의 추가 및 삭제에 용이
- 단점
	- 노드와 edge수가 많은 그래프일 수 록 메모리 사용량 증가
