# 데이터베이스
데이터를 체계적으로 저장하고, 빠르게 검색/수정/삭제할 수 있게 해주는 시스템

## 데이터베이스 사용 이유
- 저장
- 조회/검색
- 수정/삭제
- 동시성
- 무결성
- 보안/권한
- 백업/복구


## 데이터베이스 특징
- 실시간 접근성
- 지속적 변화
- 동시 공용
- 내용에 의한 참조

## 데이터베이스 종류
### 관계형 데이터베이스 (RDBMS, SQL)
표(테이블) 형태로 데이터를 저장 및 관계(Join)으로 연결

특징
- 스키마 구조를 미리 정의: 컬럼/타입/제약조건
- SQL로 강력한 조회 기능 (JOIN, GROUP BY 등)
- 트랜잭션(ACID)를 강하게 보장

사용 환경
- 데이터 구조가 비교정 명확하고 규칙적일 때
- 정확성과 일관성이 중요한 서비스


### 비관계형 데이터베이스 (NoSQL)
테이블 형태의 관계형 모델이 아닌 여러가지의 모델로 데이터를 표현하는 방식(문서, 키-값 쌍, 그래프 등)

특징
- 구조가 유연
- JOIN이 약하거나 없음

사용 환경
- 스키마가 자주 바뀌는 서비스
- 사용자 프로필, 콘텐츠 메타 데이터 같은 "덩어리 데이터"


# iOS 에서 사용되는 데이터베이스
## CoreData
- 객체 그래프 + 영속화 프레임워크
- 내부는 SQLite로 저장
- Relationship(1:N, N:N) 모델링 시 강력
- Background Context로 비동기 저장 가능
- 모델 변경 시 Migration 관리 필요

## SQLite
가장 기본적인 RDB
- 사용 방식
	- sqlite3 API 사용
	- GRDB
	- FMDB
- 표준 SQL 사용 가능
- 성능이 좋고 파일 하나로 관리 용이
- 원하는 스키마/인덱스/쿼리 최적화 가능
- 모델/관계/마이그레이션 직접 관리
- CRUD 코드 라인수 증가


## Realm
- 매우 쉬운 CRUD
- 속도가 빠르고 대용량에 강함


## SwiftData
- CoreData보다 간단
- 내부적으로는 Core Data 기반
- SwiftUI와 사용할 때 더 좋음
- 구현 간결
- iOS 17 +
