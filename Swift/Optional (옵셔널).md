
Swift에서 옵셔널(Optional)이란 무엇이며, 언제 사용해야 하나요?
옵셔널 바인딩과 강제 언래핑의 차이점은 무엇인가요?  
옵셔널 체이닝의 동작 원리는 무엇이며, 어떻게 사용하나요?  
암시적 언래핑 옵셔널(Implicitly Unwrapped Optional)은 어떤 경우에 사용해야 하나요?


# Optional
- 래핑된 값 또는 값이 없음을 나타내는 타입
- 내부 구조는 enum으로 `.some(_)`과 `.none` 로 2가지 케이스 존재
- 값이 없을 수 있는 경우 사용

## 옵셔널 바인딩 vs 강제 언래핑
### 옵셔널 바인딩
: 옵셔널이 `nil` 이 아니면 안전하게 언래핑

### 강제 언래핑
: 인스턴스에 값이 포함되어있다는 것이 확실할 경우 `!`연산자를 사용하여 해당 값을 무조건 언래핑

## 옵셔널 체이닝
: 래핑된 인스턴스의 속성과 메서드에 안전하게 접근하기 위해 옵셔널 체이닝 연산자`?.`를 사용

### 동작 원리
1. `?.`를 만날 때 마다 해당 값이 `nil`인지 확인
2. `nil`이면 즉시 전체 표현식이 `nil`로 종료 (쇼트서킷)
3. `nil`이 아닐 경우 다음 프로퍼티/메서드로 계속 접근 진행

ex:
```swift
let city = user?.address?.city
user?.profile?.updateAvatar()
```


## 암시적 언래핑 옵셔널
: 내부적으로는 Optional 타입이지만 사용할 때는 자동으로 언래핑되는 것 처럼 동작

### 주요 사용처
- 초기화 직후 반드시 채워지는 프로퍼티 (ex: 일부 `@IBOutlet`)
- 초기화 순환 문제를 피하기 위한 제한적 상황
- 테스트 코드 `sut`
