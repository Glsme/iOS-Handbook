# 암호화
권한이 없는 사용자로부터 민감한 정보를 보호하기 위해 읽을 수 있는 일반 텍스트를 읽을 수 없는 암호 텍스트로 변환하는 프로세스

### 암호화 사용 목적
- 기밀성 (Confidentiality): 민감한 정보가 허가된 사용자만 접근 가능하도록 보호
- 무결성 (Integrity): 데이터가 중간에 변조되지 않았음을 보장
- 가용성 (Availability): 필요할 떄 데이터나 시스템에 접근할 수 있도록 보장

## 암호화 유형
### 대칭 암호화
- 암호화와 복호화에 동일한 키를 사용하는 방식. 
- 빠른 성능과 효율성을 가지고 있지만 키를 안전하게 관리 필요.
- ex: AES


### 비대칭 암호화
- 암호화와 복호화 시 서로 다른 키(공개키와 개인키)를 사용하는 방식
- 공개키 → 암호화 / 개인키 → 복호화
- ex: RSA

# iOS 앱 보안 방안

### Keychain 사용
- 비밀번호, 토큰 등 민감한 데이터를 저장

### 파일 암호화
- 앱에서 생성하는 파일을 암호화 가능.

### 탈옥 환경 탐지
- 탈옥 파일 존재 여부 확인
- sandbox 탈출 여부 확인

### ATS (App Transport Security) 활성화
- TLS 1.2 강제
	- 강한 암호화 방식만 선택 가능 (ex: AES-GCM, SHA-256 등)
	- Forward Secrecy 보장 (서버의 개인키가 나중에 해킹되어도 과거 통신 내용은 복호화 불가능)
