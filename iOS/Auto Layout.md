# Auto Layout
: 앱 화면을 다양한 기기와 해상도에서 일관성있게 보여주기 위한 시스템

## 장점
- 동적 레이아웃: 화면 크기나 방향을 바꾸어도 레이아웃이 적절하게 조정
- 다양한 기기 대응: 아이폰, 아이패드와 같은 여러 기기에서도 인터페이스를 이관되게 표현

## 제약조건의 우선순위 동작
### 우선순위
- 각 제약조건이 레이아웃이 얼마나 강하게 적용될지 결정하는 요소
- 1 ~ 1000까지의 값으로 설정 (1000이 필수로 적용되는 제약 조건)

### 우선 순위 동작
- 우선순위가 높은 제약 조건이 우선적으로 적용
- 제약 조건이 서로 충돌 시  우선순위가 낮은 제약 조건은 무시

## Intrinsic Content Size
: 뷰의 콘텐츠에 따라 정해지는 기본 크기

### 활용 방법
`UILabel`, `UIButton` 같은 요소들은 텍스트 내용에 따라 고유한 크기를 가지기 때문에 
별도의 제약 조건을 주지 않아도 기본 크기로 표시



## Ambiguous Layout과 Unsatisfiable Constraints
### Ambiguous Layout
- 뷰의 위치나 크기가 모호하여 정학히 결정되지 않은 상태
- 필요한 제약 조건이 부족하거나 서로 모순되지 않지만 충분하지 않은 제약조건이 있을 때 발생
- 필수 제약 조건을 추가하여 해결
- XCode의 디버깅 기능 중 `Resolve Auto Layout issues` 를 통해 문제 확인 가능

### Unsatisfiable Constraints
- 제약 조건들이 서로 충돌하여 해결이 불가능한 상태
- 상반된 제약 조건이 동시에 적용되어 있거나 논리적으로 충돌하는 제약조건이 설정되어있을 때 발생
- 충돌하는 제약 조건을 확인 후 우선 순위를 조정하거나 불필요한 제약 조건 제거
