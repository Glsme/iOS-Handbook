# Array와 List의 차이점


| 항목       | Array (배열)          | List (리스트)                              |
| -------- | ------------------- | --------------------------------------- |
| 크기       | 보통 생성 시 크기 고정       | 동적으로 크기 변경 가능                           |
| 메모리 구조   | 연속된 메모리 공간 사용       | 구현에 따라 비연속 가능 (예: 연결 리스트)               |
| 인덱스 접근   | 매우 빠름 (`O(1)`)      | 구현마다 다름 (배열 기반 리스트는 `O(1)`, 연결 리스트는 느림) |
| 중간 삽입/삭제 | 비효율적 (`O(n)`)       | 배열 기반은 `O(n)`, 연결 리스트는 위치만 알면 유리        |
| 메모리 사용량  | 상대적으로 적음            | 포인터/노드 관리로 추가 메모리 필요 가능                 |
| 사용 목적    | 고정 길이 데이터, 빠른 랜덤 접근 | 가변 길이 데이터, 삽입/삭제가 잦은 경우                 |
| 타입 제약    | 언어에 따라 동일 타입만 저장    | 언어/구현에 따라 유연하거나 제네릭 지원                  |

> 참고: `List`는 추상 자료형(ADT)이고, `ArrayList`처럼 배열 기반으로 구현될 수도 있고 `LinkedList`처럼 연결 구조로 구현될 수도 있습니다.
